# Testing DDoS Filters with hping3 on Kali Linux

This guide details how to use the created eBPF/XDP filter on a Kali Linux system to block common volumetric attacks generated by `hping3`.

## Prerequisites (Kali Linux)

You must install the BPF Compiler Collection (BCC) tools and the linux headers that match your current kernel. 

Open a terminal in your Kali Linux Virtual Machine and run:
```bash
sudo apt update
sudo apt install bpfcc-tools linux-headers-$(uname -r) python3-bpfcc
```

## Running the Protection

On your "Target" Machine (or the machine you want to protect), ensure `xdp_filter.c` and `load_xdp.py` are in the same directory.

1. **Find your network interface name:** Run `ip a` (it might be `eth0`, `ens33`, etc).
2. **Start the filter:**
```bash
# Replace 'eth0' with your actual interface name
sudo python3 load_xdp.py eth0
```

The script will compile the C code, attach it to your network card, and sit waiting for packets.

## Simulating Attacks

From a *second* machine (or another VM acting as the attacker), use `hping3` to generate attacks against your target's IP address.

### 1. ICMP Flood (Ping Flood)
This sends massive amounts of Ping requests. The XDP filter is configured to drop ALL ICMP traffic to protect the host.

**Attacker Command:**
```bash
sudo hping3 -1 --flood <TARGET_IP>
```
*Expected Result on Target:* The python script will output `ICMP: [Number] packets dropped.`

### 2. UDP Flood
This sends massive amounts of UDP packets. The XDP filter drops all UDP traffic (Note: in reality you would whitelist DNS port 53).

**Attacker Command:**
```bash
sudo hping3 -2 --flood <TARGET_IP>
```
*Expected Result on Target:* The python script will output `UDP: [Number] packets dropped.`

### 3. Basic TCP SYN Flood (Targeting Port 80)
This attempts to exhaust server resources by initiating thousands of half-open TCP connections on the HTTP port.

**Attacker Command:**
```bash
sudo hping3 -S -p 80 --flood <TARGET_IP>
```
*Expected Result on Target:* The python script will output `TCP: [Number] packets dropped.`

## Stopping the Protection
Simply press `Ctrl+C` in the terminal running `load_xdp.py`. The script will cleanly detach the XDP program from your network interface, and normal network traffic will resume.
